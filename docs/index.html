Different tool · Same session

A macOS menu-bar app that picks up a Claude Code, Codex, or OpenCode conversation and continues it in one of the others — history, tool calls, and all.

Get it on GitHub See how it works

How it works

Three clicks from one agent to another.

Continuo reads each tool's native session files, converts the conversation into the target's own format, and opens your terminal already resumed into it. Nothing syncs in the background — sessions move only when you ask.

Under the hood

Careful with your transcripts.

Sources are read-only

Native session files are never modified. Continuo writes only its own mirror files, tracked in a local ledger, and refuses to overwrite anything it doesn't own.

Real context budgets

Transfer sizes come from each model's actual input limits (via models.dev, refreshed weekly), so a resumed session loads instead of dying on a context error.

Tools translated

Claude's Bash becomes Codex's exec_command and OpenCode's bash — history reads natively on the other side, with tool calls paired to their outputs.

Local only

Everything runs on your Mac: file conversion, the model catalog cache, and on-device titles and summaries via Apple Intelligence. No accounts, no servers.

Settings

Configured once, then forgotten.

Homes are auto-detected. Model pairs map each Claude model to its Codex counterpart — in both directions, matched per assistant message, so a session that used several models transfers each one faithfully.
